## рҹ”җ What is KeyForge?

**KeyForge Static-Auth** is a **client-side password protection system**
built exclusively with **3 core technologies**:

```
HTML  +  Vanilla JavaScript  +  CryptoJS (PBKDF2 + AES-256)
 в”Ӯ              в”Ӯ                            в”Ӯ
 в”Ӯ         Auth Engine                   Encryption
 в”Ӯ         UI / UX                       Decryption
 в”Ӯ         Session Logic                 Key Derivation
 в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
                          вҶ“
             Zero Backend. Zero Database.
             Zero Server. Zero Cost.
```

### What You Do NOT Need:

```
вқҢ Node.js / Express server        вқҢ HTTP Cookies
вқҢ PHP / Python backend            вқҢ localStorage / sessionStorage
вқҢ MySQL / MongoDB database        вқҢ Server-side sessions
вқҢ REST API endpoints              вқҢ Paid hosting
```

### Platforms That Work Out of the Box:

```

вҶ’ Same user experience вҖ” transparent upgrade

в”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғ
```

### рҹ‘Ҙ Feature 7 вҖ” Multi-User Access

```
JSON Structure:
в”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғ
{

    { "username": "DEVELOPER", "ciphertext": "salt:iv:ct..." },

Authentication Logic:
в”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғ
1. Find entry where username matches (case-insensitive)
2. If no match вҶ’ DENY immediately
3. If match вҶ’ attempt PBKDF2+AES decrypt with password
4. If decrypt fails вҶ’ DENY (wrong password)
5. If decrypt succeeds вҶ’ UNLOCK with that user's content

Security:
в”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғ
вң… Username must match (not just password)
вң… Cross-access impossible by design
вң… Each user has independently encrypted content
вң… One user's password cannot unlock another's content
```

## вқ“ FAQ

**Q: Is KeyForge actually secure?**
```
Yes вҖ” with a strong password, for intended use cases.
PBKDF2+AES-256 is industry standard.
NOT for: banking, medical, legal, classified data.
```

**Q: I forgot my password вҖ” can I recover it?**
```
No вҖ” by design. Re-encrypt with new password.
Always keep a backup of your original plaintext content!
```

**Q: Is encrypted-content.json safe to commit publicly?**
```
Yes вҖ” 100% safe. Contains only ciphertext.
Without the password it reveals nothing.
```

**Q: How do I update content?**
```
GUI: Encrypt tab вҶ’ fill fields вҶ’ generate вҶ’ replace JSON
Console: KeyForge.generateSingle(newContent, "password", "username")
Time: ~2 minutes
```

**Q: Do I need to open console (F12) anymore?**
```
No! v3.4 has a full GUI Encrypt tab.
F12 console still works as a backup method.
```

**Q: What content types are supported?**
```
вң… JSON     (structured sections, cards, tags)
вң… HTML     (raw HTML with KeyForge CSS)
вң… Markdown (# headings, **bold**, - lists, `code`)
вң… Text     (plain text, auto paragraphs)
```

**Q: Do logs get sent to a server?**
```
Never. RAM only. Refresh = completely gone.
Zero network requests beyond the JSON file.
```

**Q: Can I use KeyForge offline?**
```
Yes вҖ” self-host CryptoJS for full offline:
<script src="crypto-js.min.js"></script>
```

**Q: What is the рҹ”Ҙ button?**
```
Privacy Nuke вҖ” inspired by DuckDuckGo.
One click: clears session, content, log, returns to lock screen.
Useful before handing device to someone.
```

## вҡ пёҸ Known Limitations

```
1. JavaScript Required
   вҶ’ JS off = visual lock screen only
   вҶ’ Content still encrypted in JSON

2. Offline Brute Force Possible
   вҶ’ Someone can download JSON + try passwords locally
   вҶ’ PBKDF2 makes this 10,000x slower than plain AES
   вҶ’ Use 16+ character strong passwords

3. Single Page Only
   вҶ’ One index.html = one protected page
   вҶ’ Workaround: sections[] or HTML mode
   вҶ’ Multi-page planned for future

4. Manual Content Updates
   вҶ’ Each change needs re-encryption
   вҶ’ v3.4 GUI makes this much faster
   вҶ’ No CMS integration yet

5. No Password Recovery
   вҶ’ Lost password = lost content
   вҶ’ Always backup plaintext content
   вҶ’ Use a password manager

6. Internet Explorer
   вҶ’ Not supported (any version)
   вҶ’ All modern browsers work fine
```
